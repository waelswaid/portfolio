otel-collector container fails to start — debugging log
=======================================================

PROBLEM:
  docker compose reports "Error dependency otel-collector failed to start"
  even though the collector itself is healthy and running correctly.
  Its logs show "Everything is ready. Begin running and processing data."

ROOT CAUSE:
  The otel/opentelemetry-collector-contrib:0.120.0 image is a scratch-based
  image with NO shell utilities — no wget, no curl, no ls, no sh.
  The only binary in the image is /otelcol-contrib itself.

  This means ANY Docker healthcheck that relies on shell commands will fail,
  because there is nothing to execute them with.

  When the healthcheck fails, Docker marks the container as "unhealthy".
  Other services with `depends_on: otel-collector: condition: service_healthy`
  then refuse to start because they're waiting for a healthy status that
  can never be achieved.

WHAT WAS TRIED:
  1. healthcheck using wget  → "executable file not found in $PATH"
  2. healthcheck using curl  → "executable file not found in $PATH"
  3. healthcheck using CMD-SHELL with bash tcp check → no shell available
  4. removed healthcheck entirely and changed depends_on condition from
     service_healthy to service_started → still getting the error

  The last attempt (removing healthcheck + service_started) should work
  but docker compose may be caching the old container definition with the
  healthcheck still attached. The container was not fully recreated.
